# 第二阶段研究计划：FDD 下行 CSI 相位恢复与频域内插

> **课题名称**：《大规模 MIMO 中基于深度学习的信道状态信息反馈研究》
> **阶段定位**：第二阶段（基于第一阶段预测幅度，恢复完整下行 CSI 相位）
> **技术路线**：CNN + Transformer 级联架构（不引入 LLM）
> **生成时间**：2026-06-23
> **版本**：v2.0（含量化模块与有限反馈评估）

---

## 一、研究目标与核心问题

### 1.1 第一阶段成果

第一阶段已完成基于**上行 CSI + 历史上下行对 + 大尺度参数**的下行 CSI **幅度**预测，网络结构为 CNN + Transformer。核心结论：

- FDD 上下行大尺度衰落（路径延迟、角度、簇功率）具有强互易性，幅度预测精度高。
- FDD 上下行小尺度衰落（快衰落复增益）近似独立，**相位无法从上行 CSI 直接预测**。

### 1.2 第二阶段目标

在第一阶段预测幅度的基础上，引入**极少量下行导频反馈**（稀疏子带观测），通过深度学习频域内插恢复下行 CSI 的**相位**，最终融合得到完整的复数下行 CSI：

$$
\mathbf{H}_{\text{DL}}^{\text{pred}} = |\mathbf{H}_{\text{DL}}^{\text{stage1}}| \odot e^{j \angle \mathbf{H}_{\text{DL}}^{\text{stage2}}}
$$

### 1.3 核心科学问题

1. FDD 上下行小尺度衰落近似独立，导致相位无法从上行直接预测，但同一衰落信道的**频域相位具有高度相关性**（由多径时延结构决定）。
2. 如何利用最少数量的下行子带观测，结合第一阶段预测幅度作为先验，实现高精度相位恢复？
3. 在**有限反馈比特**约束下（实际物理信道带宽有限），量化误差对相位恢复性能的影响及鲁棒性设计。

## 二、整体技术路线

采用**两阶段级联架构**：

```
阶段一（冻结 / 轻量微调）：
  上行 CSI + 历史 CSI(16 时隙) + 大尺度参数
  ──→ 下行幅度 |H_DL|
                          ↓
阶段二（核心训练对象）：
  稀疏下行子带 CSI(含真实幅度+相位) ──┐
  阶段一预测幅度 |H_DL| ─────────────┼→ 相位恢复网络 ──→ 相位 ∠H_DL
  上行 CSI(频域能量引导) ─────────────┘
                                      ↓
                    融合：|H_DL| ⊙ e^{j∠H_DL} → 完整下行 CSI H_DL
```

## 三、数据生成方案

### 3.1 信道模型与参数

| 参数 | 配置 |
|------|------|
| **仿真平台** | Sionna 2.0 (TensorFlow) |
| **信道模型** | 3GPP TR38.901 CDL (CDL-A, CDL-C, CDL-D) |
| **场景** | UMa / UMi / RMa |
| **天线配置** | BS: 32/64 阵元 ULA；UE: 4/8 阵元 ULA |
| **OFDM 参数** | 子载波间隔 15 kHz，有效子载波数 64（可扩展至 128） |
| **带宽** | 约 0.96 MHz (64 子载波) / 1.92 MHz (128 子载波) |
| **载频** | 上行 1.95 GHz，下行 2.14 GHz（FDD 双工间隔 190 MHz） |

### 3.2 上下行信道生成策略

**物理一致性保证**：

- 上下行共享**路径延迟 $\tau$**、**离开角 AoD**、**到达角 AoA**、**簇功率 powers**
- 上下行**快衰落复增益独立采样**（体现 FDD 小尺度去相关特性）
- 显式提取 Sionna CDL 对象内部几何参数（`._aod`, `._aoa`, `._powers`, `._tau`）手动重构上下行信道

**数据格式转换**：

- Sionna (TensorFlow) → NumPy → PyTorch Tensor
- 保存为 HDF5 格式，避免训练时实时调用 TF

### 3.3 样本规模

| 数据集 | 样本数 | 说明 |
|--------|--------|------|
| 训练集 | 80,000 | 覆盖多种场景、速度、LOS/NLOS |
| 验证集 | 10,000 | 用于超参数调优和早停 |
| 测试集 | 10,000 | 严格隔离，仅用于最终评估 |

**每个样本包含**：

- 当前时隙上行 CSI：$\mathbf{H}_{\text{UL}} \in \mathbb{C}^{N_{\text{bs}} \times N_{\text{ue}} \times M}$
- 当前时隙下行 CSI（真值）：$\mathbf{H}_{\text{DL}} \in \mathbb{C}^{N_{\text{bs}} \times N_{\text{ue}} \times M}$
- 历史 16 时隙上下行 CSI 序列
- 大尺度参数向量：路径损耗、阴影衰落、RMS 延迟扩展、RMS 角度扩展、簇数量、LOS 指示

## 四、数据预处理

### 4.1 角延迟域变换

**核心处理**：将空间-频域 CSI 转换到 Angle-Delay (AD) 域，利用频域相关性。

```
空间-频域 H[f] ──→ 沿天线维度 2D-DFT ──→ 角度域
              ──→ 沿频率维度 IFFT ──→ 延迟域
              ──→ 取能量最强前 K 个角度-延迟簇
```

**参数**：

- 角度域分辨率：与天线数相同（32/64）
- 延迟域分辨率：64 点（可截断至有效多径数）
- 输出稀疏表示：$\mathbf{H}_{\text{AD}} \in \mathbb{C}^{K_{\text{ang}} \times K_{\text{ang}} \times K_{\tau}}$

### 4.2 稀疏子带采样策略

采用**自适应均匀基底 + 非均匀加密**的混合采样：

#### 4.2.1 均匀基底

由 RMS 延迟扩展自适应决定：

| 场景 | RMS 延迟扩展 | 基底间隔 | 基础采样数 |
|------|-------------|---------|-----------|
| 室内 UMi | < 100 ns | 每 16 子载波 | 4 (6.2%) |
| 典型 UMa | 100–350 ns | 每 8 子载波 | 8 (12.5%) |
| 室外 RMa | > 350 ns | 每 4 子载波 | 16 (25.0%) |

#### 4.2.2 非均匀加密

- 基于上行 CSI 频域能量峰值检测
- 在能量前 3 个峰值附近 $\pm 2$ 子载波范围内额外采样
- 加密后总采样数：约 7–19 个子载波（10%–30%）

#### 4.2.3 采样掩码

- 生成二值掩码矩阵 $\mathbf{M} \in \{0,1\}^{M}$，采样位置为 1
- 未采样位置在输入网格中初始化为 0（或第一阶段预测幅度）

#### 4.2.4 训练时数据增强

为了让网络对欠采样鲁棒，训练时**随机化采样模式**：

- 随机选择基底间隔（4/8/16）
- 随机偏移起始点（0 至 interval-1）
- 随机决定是否启用非均匀加密（概率 70%）
- **目的**：增强网络对不同采样模式的泛化能力，抗混叠学习

## 五、网络架构设计

第二阶段采用 **CNN + Transformer** 结构（不引入 LLM），与第一阶段技术栈一致。

### 5.1 输入层

| 输入 | 维度 | 来源 |
|------|------|------|
| 稀疏下行子带 CSI | $[B, N_{\text{bs}}, N_{\text{ue}}, N_{\text{sample}}]$ | 下行导频观测（经量化/反量化） |
| 采样掩码 | $[B, M]$ | 指示采样位置 |
| 第一阶段预测幅度 | $[B, N_{\text{bs}}, N_{\text{ue}}, M]$ | 阶段一输出（冻结） |
| 上行 CSI | $[B, N_{\text{bs}}, N_{\text{ue}}, M]$ | 频域能量引导 |
| 大尺度参数 | $[B, D_{\text{lsp}}]$ | 自适应采样密度控制 |

### 5.2 核心模块

#### 模块 A：频域稀疏编码器 (FreqSparseEncoder)

- **结构**：1D-CNN + LayerNorm + GELU
- **输入**：将稀疏子带 CSI 展平为 $[B, N_{\text{bs}} \cdot N_{\text{ue}}, N_{\text{sample}}]$
- **作用**：提取采样点的局部频域特征

#### 模块 B：幅度引导注意力 (Magnitude-Guided Attention)

- **结构**：Cross-Attention（Multi-Head）
- **Query**：稀疏频域特征
- **Key/Value**：第一阶段预测幅度（作为相位重要性权重）
- **机制**：幅度大的路径，其相位估计误差对整体 CSI 影响更大，网络自动分配更高注意力

#### 模块 C：频域内插网络 (Frequency Interpolator)

- **结构**：4–6 层 Transformer Encoder
- **位置编码**：正弦/余弦频域位置编码 + 可学习的采样位置嵌入
- **关键设计**：在频域维度上做自注意力，学习非采样位置的相位插值
- **替代方案**：Loewner Interpolation (LI) 模块，利用有理函数逼近频域响应

#### 模块 D：结构先验融合 (Structural Prior Fusion)

- 将上行 CSI 经轻量 CNN 编码后，与内插特征通过**门控机制**融合
- **目的**：利用上行 CSI 的角度-延迟域结构信息，抑制频域混叠

#### 模块 E：相位输出头 (Phase Head)

- **结构**：2 层 MLP
- **输出**：相位角的 $\cos/\sin$ 表示（避免 $2\pi$ 缠绕问题）
- 即输出 $\cos(\angle \mathbf{H})$ 和 $\sin(\angle \mathbf{H})$，而非直接输出角度

### 5.3 输出层与融合

$$
\angle \mathbf{H}_{\text{pred}} = \text{atan2}(\sin_{\text{out}}, \cos_{\text{out}})
$$

$$
|\mathbf{H}_{\text{pred}}| = \text{阶段一输出（冻结）}
$$

$$
\mathbf{H}_{\text{pred}} = |\mathbf{H}_{\text{pred}}| \odot e^{j \angle \mathbf{H}_{\text{pred}}}
$$

## 六、量化模块

### 6.1 量化必要性

在 5G NR 实际系统中，UE 反馈 CSI 的物理上行控制信道（PUCCH）或共享信道（PUSCH）**带宽极其有限**：
- PUCCH 格式 2/3/4 每子帧仅几十到几百比特
- 即使使用 PUSCH，为控制信令开销，CSI 反馈也必须经过**量化压缩**

如果不考虑量化，意味着 UE 以浮点精度反馈稀疏子带 CSI，这在实际中**不可能实现**。量化是"理想性能"与"实际性能"的分界线，也是评估方案工程价值的核心指标。

### 6.2 量化流程

```
UE 侧：
  估计稀疏子带 CSI (复数浮点)
        ↓
  分离：幅度（可选粗略反馈）+ 相位（必须量化反馈）
        ↓
  量化器 → 比特流 → 物理信道传输 (PUCCH/PUSCH)
        ↓
基站侧：
  接收比特流 → 反量化器 → 恢复浮点稀疏 CSI
        ↓
  输入第二阶段相位恢复网络
```

### 6.3 量化方案设计

#### 方案 A：标量量化（Scalar Quantization）—— 基准方案

- **对象**：稀疏子带 CSI 的实部与虚部分别量化（或幅度/相位分别量化）
- **比特分配**：
  - 实部：8 bit（均匀量化，范围 $[-1, 1]$）
  - 虚部：8 bit（均匀量化，范围 $[-1, 1]$）
  - 每复数元素：16 bit
- **总反馈量**：$N_{\text{sample}} \times 16$ bit
- **优点**：简单，与现有 5G NR Type I/II 码本兼容
- **缺点**：未利用子载波间相关性，比特效率低

#### 方案 B：差分量化（Differential Quantization）—— 推荐方案

- **核心思想**：相邻子载波相位差通常很小（频域相位由时延决定，变化缓慢）
- **比特分配**：
  - 首个子载波相位：8 bit（绝对相位）
  - 后续子载波相位差：4–6 bit（差分编码）
  - 幅度：2–4 bit（粗略，因基站已有第一阶段预测幅度）
- **总反馈量**：约 $N_{\text{sample}} \times (8 + 6)$ bit（首子载波 8 bit，其余 6 bit）
- **优点**：大幅降低比特数，适合频域相关结构
- **缺点**：需设计差分编解码流程，首子载波误差会传播

#### 方案 C：基于深度学习的量化/反量化（End-to-End）—— 前沿方案

- **UE 侧**：轻量 1D-CNN 量化编码器，将稀疏 CSI 压缩为低维比特表示（如 32–64 bit）
- **基站侧**：反量化解码器恢复稀疏 CSI，与第二阶段网络**联合训练**
- **优点**：比特数最少（可达传统方法的 1/10），端到端优化
- **缺点**：UE 侧增加计算复杂度，需考虑推理时延；训练不稳定（需 Gumbel-Softmax 或 Straight-Through Estimator）

### 6.4 量化感知训练（Quantization-Aware Training, QAT）

为增强网络对量化噪声的鲁棒性，训练时模拟量化误差：

**训练策略**：
- 80% 的 batch 使用**量化后**的稀疏 CSI 作为输入（模拟实际系统）
- 20% 的 batch 使用**理想浮点**稀疏 CSI 作为输入（保持网络对理想数据的拟合能力）

**量化噪声模型**（以标量量化为例）：
- 确定量化步长 $\Delta = \frac{2 \cdot \max(|\text{CSI}|)}{2^{n_{\text{bits}}}}$
- 量化：$\text{CSI}_{\text{q}} = \text{round}(\text{CSI} / \Delta) \cdot \Delta$
- 添加微小高斯噪声模拟信道传输误差：$\text{CSI}_{\text{in}} = \text{CSI}_{\text{q}} + \mathcal{N}(0, (0.1\Delta)^2)$

### 6.5 反馈开销计算

| 场景 | 采样数 | 标量量化 (16 bit/元素) | 差分量化 (平均 6 bit/元素) | 学习量化 (32–64 bit 总) |
|------|--------|----------------------|--------------------------|----------------------|
| 室内 UMi | ~8 | 128 bit | ~48 bit | 32–64 bit |
| 典型 UMa | ~12 | 192 bit | ~72 bit | 32–64 bit |
| 室外 RMa | ~18 | 288 bit | ~108 bit | 32–64 bit |

**对比参考**：传统 5G NR Type II 码本反馈开销约为 1000–4000 bit（取决于天线数和压缩比）。

## 七、损失函数

采用**多任务组合损失**：

$$
\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{MSE}} + \lambda_1 \mathcal{L}_{\text{phase}} + \lambda_2 \mathcal{L}_{\text{angle}} + \lambda_3 \mathcal{L}_{\text{consistency}} + \lambda_4 \mathcal{L}_{\text{quant}}
$$

### 各分项定义

| 损失项 | 公式 | 权重 | 作用 |
|--------|------|------|------|
| **完整 CSI MSE** | $\| \mathbf{H}_{\text{pred}} - \mathbf{H}_{\text{true}} \|_2^2$ | 1.0 | 主监督信号 |
| **相位加权 MSE** | $\| (\angle \mathbf{H}_{\text{pred}} - \angle \mathbf{H}_{\text{true}}) \odot |\mathbf{H}_{\text{stage1}}| \|_2^2$ | 0.5 | 幅度引导：强径相位误差惩罚更大 |
| **角度域 L1** | $\| \text{abs}(\text{FFT}(\mathbf{H}_{\text{pred}})) - \text{abs}(\text{FFT}(\mathbf{H}_{\text{true}})) \|_1$ | 0.1 | 保持频域稀疏结构 |
| **时序一致性** | $\| \mathbf{H}_{\text{pred}}^{(t)} - \mathbf{H}_{\text{pred}}^{(t-1)} \|_2^2$（可选） | 0.05 | 利用历史时序平滑性 |
| **量化一致性** | $\| \mathbf{H}_{\text{pred}} - \mathbf{H}_{\text{pred}}^{\text{q}} \|_2^2$（QAT 时） | 0.1 | 约束量化前后网络输出一致 |

**相位差处理**：使用

$$
\Delta\theta = \text{atan2}\left( \sin(\theta_{\text{pred}} - \theta_{\text{true}}), \cos(\theta_{\text{pred}} - \theta_{\text{true}}) \right)
$$

避免 $2\pi$ 缠绕。

## 八、训练策略

### 8.1 分阶段训练

#### 阶段 A：Warmup（Encoder 对齐）

| 配置 | 详情 |
|------|------|
| 冻结 | 第一阶段网络、频域内插网络 Transformer 层 |
| 训练 | 频域稀疏编码器、幅度引导注意力、输出头 |
| 学习率 | $1 \times 10^{-3}$ |
| Epoch | 15 |
| 目的 | 让新层输出特征与后续网络兼容 |

#### 阶段 B：主训练（端到端，理想数据）

| 配置 | 详情 |
|------|------|
| 冻结 | 第一阶段网络（完全冻结） |
| 训练 | 第二阶段所有模块 |
| 学习率 | $5 \times 10^{-4}$，带 Cosine Warmup（前 10% 步数线性升温） |
| Epoch | 60 |
| 梯度累积 | 4–8 步（模拟大 batch size） |
| 早停 | 验证集 NMSE 连续 10 个 epoch 不下降则停止 |
| 数据 | 100% 理想浮点稀疏 CSI（建立性能上界） |

#### 阶段 C：量化感知训练（QAT，实际数据）

| 配置 | 详情 |
|------|------|
| 冻结 | 第一阶段网络 |
| 训练 | 第二阶段全部模块 + 反量化解码器（若使用学习量化） |
| 学习率 | $2 \times 10^{-4}$（较低，防止破坏已收敛权重） |
| Epoch | 30 |
| 数据 | 80% 量化后稀疏 CSI + 20% 理想浮点稀疏 CSI |
| 量化方案 | 交替使用标量量化 / 差分量化 / 学习量化 |

#### 阶段 D：联合精调（可选）

| 配置 | 详情 |
|------|------|
| 解冻 | 第一阶段最后 1–2 层 + 第二阶段全部 |
| 学习率 | $1 \times 10^{-5}$（极低） |
| Epoch | 10 |
| 目的 | 微调级联边界，但需监控第一阶段幅度性能是否退化 |

## 九、评估指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **NMSE** | $\mathbb{E}\left[ \frac{\| \mathbf{H}_{\text{pred}} - \mathbf{H}_{\text{true}} \|_F^2}{\| \mathbf{H}_{\text{true}} \|_F^2} \right]$ | 主指标 |
| **余弦相似度** | $\frac{\text{Re}\{ \langle \mathbf{h}_{\text{pred}}, \mathbf{h}_{\text{true}} \rangle \}}{\| \mathbf{h}_{\text{pred}} \| \| \mathbf{h}_{\text{true}} \|}$ | 评估相位方向一致性 |
| **相位 NMSE** | NMSE 仅计算相位部分 | 专门评估相位恢复质量 |
| **频谱效率 (SE)** | $\log_2 \det(\mathbf{I} + \text{SNR} \cdot \mathbf{H}\mathbf{H}^H)$ | 系统级端到端指标 |
| **导频开销** | 采样子载波数 / 总子载波数 | 频域效率指标 |
| **反馈开销** | 总反馈比特数 / 子帧 | 实际上行信道占用指标 |
| **比特-NMSE 曲线** | 不同反馈比特数下的 NMSE | 量化鲁棒性评估 |

## 十、基线对比方案

### 10.1 理想性能基线（无量化）

## 十一、有限反馈评估

### 11.1 评估框架

有限反馈评估的核心是回答：**在相同的反馈比特预算下，本方案相比传统方法能带来多少性能增益？**

**评估维度**：

1. **固定比特预算，对比 NMSE**：在给定总反馈比特数（如 100 bit、200 bit）下，对比各方案的 NMSE。
2. **固定 NMSE 目标，对比比特数**：在达到相同 NMSE（如 -10 dB）时，对比所需的最小反馈比特数。
3. **比特-NMSE 曲线**：绘制 NMSE 随反馈比特数变化的曲线，展示量化鲁棒性。

### 11.2 关键实验设计

#### 实验 1：量化方案对比（固定采样密度）

- **条件**：典型 UMa 场景，12 个采样子载波，理想浮点 vs 标量量化 vs 差分量化 vs 学习量化
- **输出**：NMSE、余弦相似度、相位 NMSE
- **目的**：验证不同量化方案对相位恢复的影响

#### 实验 2：比特-性能权衡曲线

- **条件**：变化反馈比特数（32 → 64 → 128 → 256 → 512），固定场景
- **输出**：NMSE-比特曲线、SE-比特曲线
- **目的**：找到"性能饱和点"（即继续增加比特收益很小的临界点）

#### 实验 3：与传统 Type II 码本公平对比

- **条件**：传统 Type II 码本与本方案使用**相同反馈比特预算**
- **输出**：NMSE、SE、预编码增益
- **目的**：证明在相同开销下，本方案显著优于标准

#### 实验 4：量化误差传播分析

- **条件**：分别测试仅相位量化、仅幅度量化、联合量化的影响
- **输出**：各配置下的 NMSE 分解（幅度误差 vs 相位误差）
- **目的**：验证"幅度由阶段一预测，仅需反馈相位"策略的有效性

### 11.3 预期结果

| 场景 | 理想 (无量化) | 标量量化 (16 bit) | 差分量化 (6 bit) | 学习量化 (32 bit) |
|------|-------------|------------------|----------------|-----------------|
| 室内 UMi | NMSE = -12 dB | NMSE = -11.5 dB | NMSE = -10.8 dB | NMSE = -11.2 dB |
| 典型 UMa | NMSE = -10 dB | NMSE = -9.5 dB | NMSE = -8.8 dB | NMSE = -9.2 dB |
| 室外 RMa | NMSE = -8 dB | NMSE = -7.2 dB | NMSE = -6.5 dB | NMSE = -6.9 dB |

**关键结论**：
- 标量量化（16 bit）性能损失 < 1 dB，证明网络对量化噪声具有强鲁棒性。
- 差分量化（6 bit）在典型场景下仅损失约 1.2 dB，总反馈量仅 72 bit，远低于传统 Type II 码本。
- 学习量化（32 bit 总）性能接近标量量化，但比特数减少 75%，展示端到端优化的潜力。

## 十三、预期成果与创新点

### 13.1 预期性能

#### 理想性能（无量化）

- 在 **12.5%** 下行导频开销（典型 UMa 场景）下，完整 CSI 的 NMSE 相比"仅幅度"方案提升 **5–8 dB**
- 相位恢复 NMSE 达到 $-5$ 至 $-8$ dB（取决于场景）
- 频谱效率达到 TDD 上界的 **85–90%**

#### 有限反馈性能（有量化）

- 在 **72 bit 差分量化反馈**（典型 UMa）下，完整 CSI NMSE 相比理想性能损失 **< 1.5 dB**
- 在相同反馈比特预算（如 200 bit）下，相比 5G NR Type II 码本提升 **3–5 dB**
- 总开销（导频 + 反馈）相比传统全反馈降低 **70% 以上**

### 13.2 核心创新点

1. **幅度引导相位恢复**：首次将跨频幅度预测与频域相位内插解耦，利用第一阶段预测幅度作为相位恢复的结构性先验。
2. **自适应混合采样**：基于大尺度参数和上行频域能量的联合采样策略，实现"场景自适应"的导频密度。
3. **量化感知级联训练**：提出"理想训练 → 量化微调"的两阶段训练范式，使网络在有限反馈比特下仍保持高鲁棒性。
4. **两阶段低开销框架**：形成完整的"零反馈幅度预测 + 少反馈相位恢复"的 FDD CSI 获取框架，总开销相比传统全反馈降低 **70% 以上**。

## 十四、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 频域混叠导致相位恢复失败 | 高 | 训练时随机采样增强 + 角度域一致性损失 + 上行结构先验融合 |
